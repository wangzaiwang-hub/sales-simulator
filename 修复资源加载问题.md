# 🔧 修复生产环境资源加载问题

## 问题描述
在生产环境中，前端尝试从 `http://localhost:3001` 加载资源，导致地图和NPC不显示。

错误信息：
```
The page at https://sales-simulator-zeta.vercel.app/game was allowed to display insecure content from http://localhost:3001/resource/...
```

## 原因
前端的 `toAssetUrl` 函数没有使用环境变量 `NEXT_PUBLIC_API_URL`，而是硬编码使用本地路径。

## 修复方案

### 1. 修改代码（已完成）
修改了 `frontend/src/app/game/page.tsx` 文件：

```typescript
// 修改前
const apiUrl = "";

// 修改后
const apiUrl = process.env.NEXT_PUBLIC_API_URL || "";
```

```typescript
// 修改 toAssetUrl 函数
function toAssetUrl(path?: string) {
  if (!path) return "";
  if (/^https?:\/\//.test(path)) return path;
  
  // 移除resource/前缀（如果存在）
  const cleanPath = path.replace(/^resource\//, '');
  
  // 在生产环境中，从后端API加载资源
  if (apiUrl) {
    return `${apiUrl}/resource/${cleanPath.replace(/^\//, "")}`;
  }
  
  // 开发环境：使用前端public目录的相对路径
  return `/${cleanPath.replace(/^\//, "")}`;
}
```
