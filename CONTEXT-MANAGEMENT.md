# 智能上下文管理系统

## 概述

我们实现了一个智能的对话上下文管理系统，让 AI NPC 能够记住之前的对话内容，同时通过智能压缩避免上下文过长导致的性能问题。

## 核心功能

### 1. 对话历史记忆
- AI 现在可以看到之前的聊天记录
- 支持跨会话的对话连续性
- 自动从数据库加载历史对话

### 2. 智能上下文压缩
当对话历史过长时，系统会自动：
- **保留最近的对话**：最近 6 条消息始终保留，确保对话连贯
- **评估消息重要性**：根据多个维度评分（位置、长度、关键词、问答对）
- **选择性保留**：从旧消息中选择重要的保留
- **生成摘要**：为被丢弃的消息生成简洁摘要

### 3. Token 管理
- 自动估算消息的 token 数量
- 默认限制：2000 tokens（可配置）
- 智能分配：系统提示 + 历史摘要 + 重要消息 + 最近消息

### 重要性评分算法

消息重要性基于以下因素：
1. **位置权重**（30分）：越新的消息越重要
2. **长度权重**（15分）：较长的消息可能包含更多信息
3. **关键词权重**（15分）：包含重要关键词的消息
4. **问答权重**（10分）：问题和回答都很重要

关键词列表：
- 个人信息：名字、职业、兴趣、爱好、家人、朋友
- 重要事项：重要、记住、一定、必须、承诺、约定
- 情感相关：喜欢、讨厌、害怕、梦想、目标、秘密、感情

### 压缩策略

```
总上下文 = 系统提示 + 历史摘要 + 重要旧消息 + 最近6条消息
```

示例：
```
[系统提示] 你是 NPC 小明...
[历史摘要] 之前的对话要点：我是程序员；喜欢打游戏；工作很忙
[重要消息] 用户: 我最近在学 Python
[重要消息] NPC: 太好了！我也在学...
[最近消息] 用户: 你今天在做什么？
[最近消息] NPC: 我在整理代码...
[当前消息] 用户: 能教我一些技巧吗？
```

## 使用方式

### 后端 API

聊天接口会自动加载和使用历史上下文，无需额外配置：

```typescript
// POST /api/game/chat-with-npc
{
  "npcId": "user-npc-xxx",
  "message": "你还记得我们昨天聊的吗？"
}
```

系统会自动：
1. 加载该用户与 NPC 的聊天历史
2. 评估消息重要性
3. 压缩上下文（如果需要）
4. 生成带上下文的回复

### 配置参数

在 `contextManager.ts` 中可以调整：

```typescript
// 最大 token 数
const maxTokens = 2000;

// 保留的最近消息数
const recentCount = 6;

// 加载的历史消息数
const maxMessages = 20;
```

- 重要性评分只计算一次

## 常见问题

### Q: AI 为什么记不住很久以前的对话？
A: 如果那条消息的重要性评分低，可能被压缩掉了。可以：
- 增加 `maxTokens` 限制
- 调整重要性评分算法
- 在关键信息中使用关键词

### Q: 如何让某些消息一定被保留？
A: 在消息中包含关键词，或者修改 `evaluateImportance` 函数增加自定义规则。

### Q: 压缩会影响性能吗？
A: 压缩算法很轻量，主要是排序和过滤操作。对于 100 条消息，处理时间 <10ms。

### Q: 可以禁用压缩吗？
A: 可以，设置一个很大的 `maxTokens` 值（如 10000），但可能导致 AI 响应变慢或超出限制。
