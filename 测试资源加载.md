## 测试步骤

### 1. 测试后端资源访问

在浏览器中访问以下URL，确认资源可以访问：

#### 测试地图瓦片
```
https://你的后端域名.railway.app/resource/CGS_RU_HouseFree/img/tilesets/CGS_Urban_A2.png
```

#### 测试角色精灵图
```
https://你的后端域名.railway.app/resource/32x32%20Customizable%20Character%20Pack/Walk/Character/Character_Walk_Front-Sheet.png
```

如果能看到图片，说明后端资源路径正确。

### 2. 测试游戏页面

访问：
```
https://sales-simulator-zeta.vercel.app/game
```

#### 应该看到：
- ✅ 地图正常显示（瓦片、建筑、装饰）
- ✅ 玩家角色正常显示（完整的身体）
- ✅ NPC 正常显示（完整的身体）
- ✅ 角色名字标签显示
- ✅ 可以使用方向键移动

#### 不应该看到：
- ❌ 黑屏
- ❌ 只有名字没有身体
- ❌ 浏览器控制台的资源加载错误

### 3. 检查浏览器控制台

按 F12 打开开发者工具，查看 Console 标签：

#### 正常情况：
- 没有 `localhost:3001` 相关的错误
- 没有 `insecure content` 警告
- 资源加载成功

#### 如果还有错误：
记录错误信息，检查：
1. 环境变量 `NEXT_PUBLIC_API_URL` 是否正确
2. Railway 后端是否正常运行
3. 后端 `resource` 目录是否正确部署

## 预期结果

修复后，所有资源应该从以下路径加载：
```
https://你的后端域名.railway.app/resource/...
```

而不是：
```
http://localhost:3001/resource/...
```

## 如果还有问题

### 检查 Vercel 环境变量

确认以下环境变量已设置：
```
NEXT_PUBLIC_API_URL=https://你的后端域名.railway.app
```

### 检查 Railway 部署

1. 访问 Railway 控制台
2. 查看 "Deployments" 日志
3. 确认 `resource` 目录已部署
4. 测试资源访问：`https://你的后端域名.railway.app/resource/test.png`

### 清除缓存

如果修复后还是有问题，尝试：
1. 清除浏览器缓存
2. 使用无痕模式访问
3. 强制刷新（Ctrl+Shift+R 或 Cmd+Shift+R）
