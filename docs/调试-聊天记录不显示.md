# 调试：聊天记录不显示

你反馈：刷新之后聊天记录还是不显示。

## 我添加的调试日志

我在后端代码中添加了详细的日志：

### 保存聊天记录时
- `💾 正在保存聊天记录到数据库...`
- `✅ 聊天记录保存成功`
- 或 `❌ Failed to save chat message: [错误信息]`

### 加载聊天历史时
- `📖 加载聊天历史: userId=xxx, targetUserId=xxx`

## 现在请测试

### 步骤1：与NPC聊天

1. 打开游戏页面
2. 与任意NPC聊天
3. 发送消息："你好"
4. 等待AI回复

### 步骤2：查看后端日志

打开终端，查看后端服务器的输出，应该看到：

```
💾 正在保存聊天记录到数据库...
✅ 聊天记录保存成功
```

**如果看到这个**：说明保存成功 ✅

**如果看到错误**：
- `❌ Failed to save chat message: relation "ChatMessage" does not exist`
  - 说明ChatMessage表不存在
  - 需要运行数据库迁移脚本

- 其他错误：截图发给我

### 步骤3：刷新页面

1. 按F5刷新页面
2. 再次打开与该NPC的聊天窗口

### 步骤4：查看后端日志

应该看到：

```
📖 加载聊天历史: userId=xxx, targetUserId=xxx
📊 从数据库查询到 X 条记录
✅ 过滤后有 X 条相关记录
```

**如果看到这个**：
- 如果"过滤后有 X 条相关记录"中X > 0，说明有记录
- 如果X = 0，说明没有找到相关记录

**如果看到错误**：截图发给我

### 步骤5：查看浏览器控制台

1. 按F12打开开发者工具
2. 切换到Console标签
3. 查看是否有错误信息

4. 切换到Network标签
5. 刷新页面并打开聊天窗口
6. 查找`chat-history`请求
7. 点击查看响应内容

**预期响应**：
```json
[
  {
    "id": "...",
    "userId": "...",
    "targetUserId": "...",
    "message": "你好",
    "reply": "...",
    "affinity": 25,
    "familiarity": 5,
    ...
  }
]
```

**如果响应是空数组 `[]`**：
- 说明数据库中没有记录
- 或者过滤条件有问题

## 可能的原因

### 原因1：ChatMessage表不存在

**症状**：
- 后端日志显示：`relation "ChatMessage" does not exist`

**解决**：
1. 打开 Supabase Dashboard
2. 进入 SQL Editor
3. 运行 `backend/migrations/add-chat-history.sql`

### 原因2：数据没有保存

**症状**：
- 没有看到"💾 正在保存聊天记录到数据库..."日志
- 或者看到保存失败的错误

**检查**：
- 查看后端日志的完整错误信息
- 截图发给我

### 原因3：查询条件有问题

**症状**：
- 后端日志显示"从数据库查询到 X 条记录"（X > 0）
- 但"过滤后有 0 条相关记录"

**可能原因**：
- userId或targetUserId不匹配
- 需要检查用户ID

### 原因4：前端没有调用API

**症状**：
- 后端没有看到"📖 加载聊天历史"日志

**检查**：
1. 打开浏览器开发者工具
2. 切换到Network标签
3. 刷新页面并打开聊天窗口
4. 查看是否有`chat-history`请求

**如果没有请求**：
- 说明前端没有调用API
- 可能是前端代码有问题
