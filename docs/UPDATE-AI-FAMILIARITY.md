# 更新：熟悉度也由AI评判

## 问题

用户反馈：好感度和熟悉度好像都是固定的，不是AI自己评判的。

## 现状分析

**之前的实现**：
- ✓ 好感度由AI评判（affinityChange: -10到+10）
- ✗ 熟悉度用固定规则计算（每次对话+2到+5）

**问题**：
- 熟悉度变化不够智能
- 无法反映对话的深度和质量
- 简短寒暄和深入交流的熟悉度增加一样多

## 解决方案

### 现在的实现

**都由AI评判**：
- ✓ 好感度由AI评判（affinityChange: -10到+10）
- ✓ 熟悉度由AI评判（familiarityChange: 0到+10）

### AI评判标准

AI会根据以下因素评判熟悉度变化：

#### 1. 对话深度

**简短寒暄**（+1到+3）：
```
用户："你好"
NPC："你好啊"
→ 熟悉度 +1
```

**一般对话**（+3到+5）：
```
用户："你最近在做什么？"
NPC："我在学习新技术"
→ 熟悉度 +4
```

**深入交流**（+5到+7）：
```
用户："你为什么对创作感兴趣？"
NPC："因为创作让我感到自由，可以表达内心的想法..."
→ 熟悉度 +6
```

**非常深入**（+8到+10）：
```
用户："我也有类似的困惑，你是怎么克服的？"
NPC："我经历过一段很艰难的时期，后来我发现..."
→ 熟悉度 +9
```

#### 2. 信息分享

- 是否分享个人信息
- 是否表达真实想法
- 是否讨论深层话题
- 是否互相了解更多

#### 3. 对话质量

- 对话是否有实质内容
- 是否建立更深的连接
- 是否产生共鸣

## 优势

### 1. 智能评判

- AI根据对话内容决定熟悉度增加多少
- 不再是固定的+2到+5
- 反映真实的对话深度

### 2. 上下文连贯

- 简短寒暄：熟悉度增加少
- 深入交流：熟悉度增加多
- 符合人类的社交逻辑

### 3. 个性化

- 不同NPC对同样对话的评判可能不同
- 外向的人可能更容易增加熟悉度
- 内向的人可能需要更深入的对话

### 4. 真实感

- 熟悉度变化有理有据
- AI会解释为什么熟悉度增加
- 更符合真实的人际关系发展

## 回退机制

如果AI评估失败，系统会使用简单的回退方案：

```typescript
// 回退方案：基于规则计算熟悉度
let familiarityChange = 2; // 基础值
if (userMessage.length > 30) familiarityChange += 1;
if (userMessage.length > 50) familiarityChange += 1;
if (topicMatch) familiarityChange += 1;
// 结果：2到5
```

但主要还是依赖AI评判，回退只是保底。

## 数据库存储

熟悉度存储在Relationship表中：

```sql
CREATE TABLE "Relationship" (
  ...
  "familiarity" INTEGER NOT NULL DEFAULT 0, -- 0到100
  ...
);
```

每次对话后：
```typescript
newFamiliarity = Math.max(0, Math.min(100, 
  currentFamiliarity + AI评判的familiarityChange
));
```
