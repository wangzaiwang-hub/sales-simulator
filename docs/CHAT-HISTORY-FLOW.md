# 聊天记录系统流程图

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户界面                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  游戏画面 (game/page.tsx)                            │  │
│  │  - 玩家移动                                          │  │
│  │  - NPC显示（带状态标签）                            │  │
│  │  - 点击NPC打开聊天窗口                              │  │
│  └──────────────────────────────────────────────────────┘  │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  聊天窗口                                            │  │
│  │  - 自动加载历史记录 ← 新功能！                      │  │
│  │  - 显示对话                                          │  │
│  │  - 显示关系变化                                      │  │
│  │  - 显示AI评估                                        │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────────┐
│                      后端API                                 │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  POST /api/game/npc-chat                             │  │
│  │  - 接收用户消息                                      │  │
│  │  - 生成NPC回复                                       │  │
│  │  - AI评估关系变化                                    │  │
│  │  - 保存到数据库 ← 已实现                            │  │
│  │  - 返回回复和关系信息                                │  │
│  └──────────────────────────────────────────────────────┘  │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  GET /api/game/chat-history ← 新API！                │  │
│  │  - 查询聊天历史                                      │  │
│  │  - 按时间排序                                        │  │
│  │  - 返回最近50条记录                                  │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────────┐
│                    Supabase数据库                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  ChatMessage 表 ← 需要运行迁移脚本创建！             │  │
│  │  ┌────────────────────────────────────────────────┐  │  │
│  │  │ id                  - 记录ID                   │  │  │
│  │  │ userId              - 发送者ID                 │  │  │
│  │  │ targetUserId        - 接收者ID（NPC）          │  │  │
│  │  │ message             - 用户消息                 │  │  │
│  │  │ reply               - NPC回复                  │  │  │
│  │  │ affinity            - 好感度                   │  │  │
│  │  │ familiarity         - 熟悉度                   │  │  │

│  │  │ familiarityChange   - 熟悉度变化               │  │  │
│  │  │ aiReasoning         - AI评判理由               │  │  │
│  │  │ aiEmotionalResponse - AI情绪反应               │  │  │
│  │  │ npcMoodBefore       - NPC对话前心情            │  │  │

│  │  │ createdAt           - 创建时间                 │  │  │
│  │  └────────────────────────────────────────────────┘  │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 聊天流程（发送消息）

```
用户在聊天窗口输入消息
         ↓
    按Enter发送
         ↓
┌────────────────────────┐
│ 1. 前端发送请求        │
│    POST /api/game/     │
│    npc-chat            │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 2. 后端处理            │
│    - 检查关系          │
│    - 判断是否接受      │
│    - 生成NPC回复       │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 3. AI评估              │
│    - 分析对话内容      │
│    - 评估好感度变化    │
│    - 评估熟悉度变化    │
│    - 生成评判理由      │
│    - 更新NPC心情状态   │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 4. 保存到数据库 ✅     │
│    INSERT INTO         │
│    ChatMessage         │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 5. 返回给前端          │
│    - NPC回复           │
│    - 关系变化          │
│    - AI评估            │
└────────────────────────┘
         ↓
    显示在聊天窗口
```

## 加载历史记录流程（新功能）

```
用户点击NPC打开聊天窗口
         ↓
┌────────────────────────┐
│ 1. 检查是否已有记录    │
│    chatMessages[npcId] │
│    ?.length > 0        │
└────────────────────────┘
         ↓
    没有记录？
         ↓ 是
┌────────────────────────┐
│ 2. 发送加载请求        │
│    GET /api/game/      │
│    chat-history?       │
│    targetUserId=npcId  │
│    &limit=50           │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 3. 后端查询数据库      │
│    SELECT * FROM       │
│    ChatMessage         │
│    WHERE ...           │
│    ORDER BY createdAt  │
│    DESC LIMIT 50       │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 4. 返回历史记录        │
│    [{                  │
│      message: "...",   │
│      reply: "...",     │
│      affinity: 28,     │
│      ...               │
│    }]                  │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 5. 前端转换格式        │
│    数据库格式 →        │
│    ChatMessage格式     │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 6. 反转顺序            │
│    history.reverse()   │
│    (最早的在前)        │
└────────────────────────┘
         ↓
┌────────────────────────┐
│ 7. 更新state           │
│    setChatMessages()   │
└────────────────────────┘
         ↓
    显示在聊天窗口 ✅
```

### 刷新后加载

```

  ↓
前端state清空
  ↓
用户再次打开与该NPC的聊天
  ↓
前端检测到没有记录
  ↓
自动调用 GET /api/game/chat-history
  ↓
从数据库加载历史记录
  ↓
转换格式并显示
  ↓
用户看到完整的聊天历史 ✅
```

## 关键点

### ✅ 已实现

1. **自动保存**：每次对话后自动保存到数据库
2. **自动加载**：打开聊天窗口时自动加载历史
3. **完整信息**：保存和显示所有关系变化和AI评估
4. **智能缓存**：避免重复加载
5. **正确排序**：按时间顺序显示

### ⚠️ 需要用户操作

1. **运行迁移脚本**：在Supabase中创建ChatMessage表
   - 文件：`backend/migrations/add-chat-history.sql`
   - 位置：Supabase Dashboard → SQL Editor

### 🔄 工作流程

```
第一次聊天 → 保存到数据库 → 显示在前端
     ↓
刷新页面 → state清空
     ↓
再次打开聊天 → 从数据库加载 → 显示历史记录
     ↓
继续聊天 → 追加新消息 → 保存到数据库
```
