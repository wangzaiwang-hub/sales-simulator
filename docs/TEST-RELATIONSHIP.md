## 测试步骤

### 测试1：陌生人拒绝

**目标**：验证社恐和内向NPC会拒绝陌生人

1. 访问 https://192.168.110.169:3443/game
2. 找到一个NPC（第一次见面）
3. 按C键或点击NPC尝试聊天
4. 观察结果：
   - ✅ 如果被拒绝，查看拒绝理由
   - ✅ 拒绝理由应该是以下之一：
     - "不好意思，我不太习惯和陌生人说话..."（社恐）
     - "抱歉，我现在不太想聊天。"（内向）
     - "我们好像不认识吧..."（神经质）
     - "不好意思，我有点事。"（开放性低）

### 测试2：初始好感度

**目标**：验证初始好感度计算

1. 成功和一个NPC聊天
2. 查看聊天窗口底部的关系信息
3. 观察：
   - ✅ 关系类型：陌生人
   - ✅ 好感度：-20 到 40 之间
   - ✅ 熟悉度：0（第一次见面）

### 测试3：好感度提升

**目标**：验证多次互动提升好感度

1. 和同一个NPC多次聊天（5-10次）
2. 每次聊天后观察关系变化
3. 观察：
   - ✅ 好感度逐渐增加（每次 +1 到 +5）
   - ✅ 熟悉度逐渐增加（每次 +2 到 +5）
   - ✅ 关系类型变化：陌生人 → 熟人 → 朋友

### 测试4：关系类型变化

**目标**：验证关系类型自动更新

继续和同一个NPC聊天，观察关系类型变化：

1. **陌生人**（初始）
   - 好感度：任意
   - 熟悉度：< 20

2. **熟人**（聊天3-5次后）
   - 熟悉度：> 20

3. **朋友**（聊天10-15次后）
   - 好感度：> 40
   - 熟悉度：> 30

4. **好友**（聊天20+次后）
   - 好感度：> 70
   - 熟悉度：> 60

### 测试5：好友不拒绝

**目标**：验证好友几乎不会拒绝聊天

1. 将关系提升到好友级别
2. 多次尝试聊天
3. 观察：
   - ✅ 几乎总是接受聊天
   - ✅ 只有在压力非常大时才会拒绝

### 测试6：不同性格的NPC

**目标**：验证不同性格NPC的行为差异

尝试和多个不同的NPC聊天：

1. **外向NPC（外向性 > 60）**
   - ✅ 更容易接受陌生人聊天
   - ✅ 初始好感度较高

2. **内向NPC（外向性 < 40）**
   - ✅ 经常拒绝陌生人
   - ✅ 需要多次互动才能建立关系

3. **宜人性高NPC（宜人性 > 60）**
   - ✅ 对所有人都比较友好
   - ✅ 好感度提升较快

4. **宜人性低NPC（宜人性 < 40）**
   - ✅ 可能随机拒绝聊天
   - ✅ 拒绝理由："不想聊。"

### 测试7：关系信息显示

**目标**：验证前端正确显示关系信息

1. 和NPC聊天
2. 查看聊天窗口
3. 观察每条NPC回复下方的关系信息：
   - ✅ 显示关系类型
   - ✅ 显示好感度（数值 + 文字 + 变化量）
   - ✅ 显示熟悉度（数值 + 文字 + 变化量）
   - ✅ 好感度有颜色编码

### 测试8：边际效应递减

**目标**：验证好感度增长递减

1. 将好感度提升到50以上
2. 继续聊天
3. 观察：
   - ✅ 好感度增长变慢（每次 +0.5 到 +2.5）
   - ✅ 从50提升到70需要更多互动

## 数据库验证

### 查看关系数据

```sql
-- 查看所有关系
SELECT 
  r.*,
  u1.username as npc_name,
  u2.username as player_name
FROM "Relationship" r
JOIN "User" u1 ON r."userId" = u1.id
JOIN "User" u2 ON r."targetUserId" = u2.id
ORDER BY r."updatedAt" DESC
LIMIT 10;
```

应该看到：
- ✅ 每次聊天都创建或更新关系记录
- ✅ affinity 和 familiarity 随互动增加
- ✅ relationshipType 自动更新

### 查看互动历史

```sql
-- 查看互动次数
SELECT 
  u.username,
  r."interactionCount",
  r.affinity,
  r.familiarity,
  r."relationshipType"
FROM "Relationship" r
JOIN "User" u ON r."userId" = u.id
WHERE r."targetUserId" = '你的用户ID'
ORDER BY r."interactionCount" DESC;
```

## 故障排查

### 问题1：所有NPC都接受聊天

**原因**：关系系统未生效

**检查**：
1. 数据库迁移是否运行
2. 后端是否重启
3. 查看后端日志是否有错误

### 问题2：看不到关系信息

**原因**：前端未更新

**检查**：
1. 前端是否重启
2. 浏览器缓存是否清除
3. 查看浏览器控制台错误

### 问题3：好感度不变化

**原因**：数据库更新失败

**检查**：
1. 查看后端日志
2. 检查Relationship表是否存在
3. 检查外键约束

### 问题4：拒绝率太高/太低

**原因**：性格参数不合理

**调整**：
```sql
-- 让NPC更外向
UPDATE "User"
SET "personalityTraits" = jsonb_set(
  "personalityTraits",
  '{extraversion}',
  '70'
)
WHERE id = 'NPC用户ID';

-- 让NPC更内向
UPDATE "User"
SET "personalityTraits" = jsonb_set(
  "personalityTraits",
  '{extraversion}',
  '25'
)
WHERE id = 'NPC用户ID';
```
