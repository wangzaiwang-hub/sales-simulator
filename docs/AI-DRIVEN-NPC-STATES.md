# AI驱动的NPC状态系统

## 概述

NPC的心情和活动状态现在完全由SecondMe AI自己决定，不再限制在固定的几个选项中。AI可以根据自己的性格、经历和感受，自由地描述当前的状态。

## 核心理念

### 之前的问题

**固定选项限制**：
- 心情只能是：happy, sad, angry, excited, tired, focused, confused, calm, anxious, neutral
- 状态只能是：idle, thinking, busy, chatting, working, resting, arguing, laughing, pondering
- 感觉像是在填表格，不够真实

**缺乏个性**：
- 所有NPC都用同样的词汇描述状态
- 无法体现个人特色和真实感受

### 新的设计

**AI自由描述**：
- 心情：AI可以用任何2-4个字的词语描述，如"充满灵感"、"有点累"、"专注平静"、"期待兴奋"等
- 状态：AI可以用任何2-5个字的词语描述，如"构思新作品"、"喝咖啡休息"、"写代码"、"四处看看"等

**真实个性化**：
- 创作者可能说"充满灵感，构思新作品"
- 程序员可能说"专注平静，写代码"
- 上班族可能说"有点累，喝咖啡休息"
- 学生可能说"好奇兴奋，学习新东西"

## 工作原理

### 1. 对话后状态更新

每次与NPC对话后，AI会根据对话内容更新状态：

```typescript
// AI评估对话
{
  "affinityChange": 3,
  "newMood": "充满期待",  // ✓ AI自己决定
  "newActivityStatus": "继续交流",  // ✓ AI自己决定
  "reasoning": "对方问到了我感兴趣的话题，而且态度很真诚。",
  "emotionalResponse": "我感到很开心，想继续聊下去。"
}
```

### 2. 定期状态更新

可以定期调用API让所有NPC更新状态：

```bash
POST /api/game/update-npc-states
```

AI会根据NPC的性格和当前状态，自主决定新的心情和活动：

```typescript
// AI更新状态
{
  "currentMood": "专注投入",  // ✓ AI自己决定
  "activityStatus": "整理创作素材",  // ✓ AI自己决定
  "reasoning": "我最近有很多创作想法，需要整理一下思路。"
}
```

**程序员**：
- 心情：专注平静、逻辑清晰、有点疲惫

- 活动：学习新知识、做作业、复习功课

**上班族**：
- 心情：有点累、需要休息、平淡如常
- 活动：喝咖啡休息、处理工作、发呆放空

**外向开朗**：
- 心情：充满活力、开心愉快、期待交流

- 心情：平静独处、沉浸思考、享受宁静
- 活动：独自思考、看书、听音乐

**认真负责**：
- 心情：专注认真、有条不紊、追求完美

**随性自由**：
- 心情：轻松自在、随遇而安、无拘无束
- 活动：随便逛逛、发呆、想到什么做什么

## API端点

### 1. 更新所有NPC状态

```
POST /api/game/update-npc-states
Authorization: Bearer <token>
```

**响应**：
```json
{
  "message": "NPC states updated",
  "updates": [
    {
      "npcId": "...",
      "npcName": "张三",
      "oldMood": "平静",
      "newMood": "充满灵感",
      "oldStatus": "闲逛",
      "newStatus": "构思新作品",
      "reasoning": "我最近有很多创作想法..."
    }
  ]
}
```

### 2. 对话自动更新

与NPC对话后，状态会自动更新，无需额外调用。

## 使用场景

### 场景1：定期更新

可以设置定时任务，每隔一段时间更新所有NPC的状态：

```javascript
// 每5分钟更新一次
setInterval(async () => {
  await fetch('/api/game/update-npc-states', {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` }
  });
}, 5 * 60 * 1000);
```

### 场景2：对话触发

每次对话后，NPC的状态会根据对话内容自动更新：

```javascript
// 对话后自动更新
const response = await chatWithNpc(npcId, message);
console.log('新状态：', response.aiEvaluation.newMood);
console.log('新活动：', response.aiEvaluation.newActivityStatus);
```

### 场景3：手动触发

玩家可以点击按钮手动刷新所有NPC状态：

```javascript
// 刷新按钮
<button onClick={updateAllNpcStates}>
  刷新NPC状态
</button>
```

## 优势

### 1. 真实感

- AI根据性格和情境自由描述
- 每个NPC都有独特的表达方式
- 状态描述更贴近真实人类

### 2. 多样性

- 不再限制在固定选项
- 状态描述千变万化
- 每次更新都可能不同

### 3. 个性化

- 创作者会说"充满灵感"
- 程序员会说"专注平静"
- 上班族会说"有点累"
- 体现职业和性格特点

### 4. 动态变化

- 对话后状态会变化
- 定期更新保持活力
- 状态反映当前情境

## 回退机制

如果AI调用失败，系统会使用简单的回退方案：

```typescript
// 回退方案：根据性格生成状态
- 外向的人：充满活力、找人聊天
- 尽责的人：专注、工作中
- 开放的人：充满灵感、创作
```

但主要还是依赖AI生成，回退只是保底。

## 数据库存储

NPC状态存储在User表中：

```sql
ALTER TABLE "User" 
ADD COLUMN "currentMood" TEXT,
ADD COLUMN "activityStatus" TEXT;
```

- `currentMood`: AI自定义的心情描述（TEXT类型，不再限制）
- `activityStatus`: AI自定义的活动状态（TEXT类型，不再限制）

## 注意事项

### 1. API调用成本

- 每次更新都会调用AI
- 建议控制更新频率（如5-10分钟一次）
- 对话后更新是必要的，定期更新是可选的

### 2. 状态长度

- 建议心情2-4个字
- 建议活动2-5个字
- 太长会影响显示效果

### 3. 状态合理性

- AI通常会给出合理的状态
- 但偶尔可能出现不太合适的描述
- 可以通过提示词优化
