# 聊天窗口头部 - 极简单行布局

## 实现状态
✅ 已完成

## 设计要求
根据用户提供的截图和反馈，实现极简的单行头部布局：
- 头部只有一行，不显示NPC名字和职业信息
- 进度条宽度受限，不占满整个宽度
- 头像靠近进度条
- 关闭按钮更大更明显

## 实现细节

### 布局结构
```
[QQ] 👤 [进度条区域] 👤 [关闭]
```

### 具体尺寸
- 头部高度：~45px（极简）
- 头像大小：36px × 36px (w-9 h-9)
- 进度条区域：max-width 180px（限制宽度）
- 关闭按钮：px-3 py-1.5（加大尺寸）
- 间距：gap-2（紧凑）
- 内边距：px-3 py-2（最小化）

### 元素说明

1. **QQ标签**（左侧）
   - 文字大小：9px
   - 颜色：slate-900/60
   - flex-shrink-0（不收缩）

2. **玩家头像**
   - 圆形头像，渐变背景（cyan-400 to blue-500）
   - 显示用户名首字母

3. **进度条区域**（中间）
   - flex-1 min-w-0 max-w-[180px]（限制最大宽度）
   - 包含两个进度条：
     - 好感度：-100到+100（红→黄→绿渐变）
     - 熟悉度：0到100（蓝→紫渐变）
   - AI回复时显示"输入中..."动画

4. **NPC头像**
   - 圆形头像，渐变背景（pink-400 to purple-500）
   - 显示NPC名字首字母

5. **关闭按钮**（右侧）
   - 加大尺寸：px-3 py-1.5
   - 文字：11px
   - 背景：white/40，hover时white/60

### 聊天区域
- 高度：55vh，最小400px
- 占据更多空间，方便查看聊天记录

## 用户体验
- 头部极简，信息密度高
- 进度条一目了然，实时显示关系变化
- AI回复时有明显的"输入中"提示
- 关闭按钮更大，更容易点击
- 聊天区域更大，阅读体验更好
