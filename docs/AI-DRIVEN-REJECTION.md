# AI驱动的拒绝理由系统

## 概述

NPC拒绝聊天时的理由现在由SecondMe AI自己生成，不再是固定的几句话。AI会根据NPC的性格、状态和拒绝原因，给出真实、自然的拒绝理由。

## 问题

**之前的实现**：
- 拒绝理由都是固定的字符串
- 所有NPC用同样的话拒绝
- 感觉很生硬，没有个性

**固定的拒绝理由**：
```typescript
"不好意思，我现在不太想聊天。"
"抱歉，我现在不太想聊天。"
"我们好像不认识吧..."
"不好意思，我有点事。"
"我现在心情不太好，不想说话。"
"我现在有点忙，等会儿再说吧。"
"我有点累了，想休息一下。"
"不想聊。"
"我不想和你说话。"
```

## 解决方案

### AI生成拒绝理由

现在NPC拒绝时，AI会根据以下信息生成个性化的拒绝理由：

1. **NPC背景**：
   - 职业
   - 兴趣
   - 个人简介
   - 性格特征

2. **当前状态**：
   - 心情
   - 正在做什么
   - 与对方的关系

3. **拒绝原因**：
   - 社恐（social_anxiety）
   - 内向（introverted）
   - 敏感警惕（nervous）
   - 保守（conservative）
   - 心情不好（bad_mood）
   - 忙碌（busy）
   - 疲惫（tired）
   - 不友善（unfriendly）
   - 敌对（enemy）
   - 随机（random）

## AI提示词

```
你是[NPC名字]，一个真实的人。有人想和你聊天，但你现在不想接受。

【你的背景】
- 职业：创作者
- 兴趣：商业、创作、科技
- 性格：外向性 30/100 (内向)，宜人性 60/100 (友善)

【当前状态】
- 心情：平静
- 正在做：思考中
- 关系：陌生人（第一次见面）

【情况】
你是一个内向的人，对陌生人比较谨慎

【任务】
请以[NPC名字]的视角，用1-2句话礼貌地拒绝对方的聊天请求。

要求：
1. 要符合你的性格特征
2. 要真实、自然，不要太生硬
3. 可以简短，也可以稍微解释一下原因
4. 不要太冷漠，保持基本的礼貌
5. 根据你的性格，可以直接一点或委婉一点
```

## 技术实现

### 1. 判断拒绝原因

```typescript
// shouldAcceptInteraction 返回拒绝类型
{
  accept: false,
  reason: "不好意思，我不太习惯和陌生人说话...", // 回退理由
  rejectionType: "social_anxiety" // 拒绝类型
}
```

### 2. 构建AI提示

```typescript
const prompt = buildRejectionPrompt(
  npcName,
  npcPersonality,
  currentState,
  {
    isFirstMeeting,
    relationshipType,
    reason: rejectionType
  }
);
```

### 3. 调用AI生成

```typescript
const aiReason = await generateRejectionWithAI(
  prompt,
  secondmeAccessToken
);
```

### 4. 回退机制

如果AI生成失败，使用多样化的回退理由：

```typescript
const fallbackReasons = {
  social_anxiety: [
    '不好意思，我不太习惯和陌生人说话...',
    '抱歉，我有点社恐，不太会聊天...',
  ],
  // ... 每种类型有多个备选
};
```

## 工作流程

```
1. 用户尝试与NPC聊天
   ↓
2. 系统判断是否接受
   ↓
3. 如果拒绝，确定拒绝原因类型
   ↓
4. 构建AI提示词（包含NPC性格、状态、拒绝原因）
   ↓
5. 调用SecondMe AI生成拒绝理由
   ↓
6. 如果AI成功，返回AI生成的理由
   ↓
7. 如果AI失败，使用多样化的回退理由
   ↓
8. 返回拒绝理由给用户
```

## 优势

### 1. 个性化

- 每个NPC的拒绝方式都不同
- 符合NPC的性格特征
- 更真实、更有代入感

### 2. 多样性

- 同一个NPC每次拒绝的话可能不同
- 不会感觉重复
- 增加游戏的可玩性

### 3. 情境化

- 根据当前状态调整拒绝理由
- 心情不好时的拒绝和忙碌时的拒绝不同
- 对陌生人和敌人的拒绝方式不同

### 4. 真实感

- 拒绝理由更自然
- 不再是机械的固定回复
- 像真人一样有不同的表达方式

## 回退机制

### 多样化的回退理由

即使AI失败，系统也会从多个备选理由中随机选择：

```typescript
// 每种拒绝类型有2个备选理由
const fallbackReasons = {
  social_anxiety: [
    '不好意思，我不太习惯和陌生人说话...',
    '抱歉，我有点社恐，不太会聊天...',
  ],
  introverted: [
    '抱歉，我现在不太想聊天。',
    '不好意思，我想一个人待会儿。',
  ],
  // ... 其他类型
};

// 随机选择一个
const reason = reasons[Math.floor(Math.random() * reasons.length)];
```

## 注意事项

### 1. API调用成本

- 每次拒绝都会调用AI
- 但拒绝频率不高，成本可控
- 可以考虑缓存最近的拒绝理由

### 2. 响应时间

- AI生成需要1-2秒
- 用户体验可接受
- 有回退机制保证不会卡住

### 3. 理由合理性

- AI通常会给出合理的拒绝理由
- 但偶尔可能不太合适
- 可以通过优化提示词改进
