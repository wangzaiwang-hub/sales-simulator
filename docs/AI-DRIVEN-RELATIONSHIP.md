# AI驱动的关系评估系统

## 概述

使用SecondMe AI来评判对话质量、好感度变化和熟悉度变化，让NPC的反应更真实、更有感情。AI会根据对话内容、NPC性格、当前关系等因素，给出有理有据的评估。

## 重要更新（2026-04-10）

### v3.2.1：熟悉度也由AI评判

**之前的问题**：
- 好感度由AI评判 ✓
- 熟悉度用固定规则计算 ✗（每次对话+2到+5）
- 熟悉度变化不够智能，无法反映对话深度

**现在的改进**：
- 好感度由AI评判 ✓
- 熟悉度也由AI评判 ✓（0到+10，根据对话深度）
- AI会根据对话内容决定熟悉度增加多少

### 熟悉度评判标准

AI会根据以下因素评判熟悉度变化：

1. **对话深度**：
   - 简短寒暄：+1到+3
   - 一般对话：+3到+5
   - 深入交流：+5到+7
   - 非常深入：+8到+10

2. **信息分享**：
   - 是否分享个人信息
   - 是否表达真实想法
   - 是否讨论深层话题

3. **对话质量**：
   - 对话是否有实质内容
   - 是否互相了解更多
   - 是否建立更深的连接

## 核心特性

### 1. AI评估好感度和熟悉度变化
- **不再是冷冰冰的脚本**：AI会真正理解对话内容
- **好感度由AI评判**：基于对话质量、话题匹配、态度等
- **熟悉度也由AI评判**：基于对话深度、信息分享、交流质量
- **考虑上下文**：基于NPC的性格、兴趣、当前关系
- **有理有据**：AI会解释为什么好感度和熟悉度增加或减少
- **情感反应**：AI会描述NPC的真实感受

### 2. 动态情绪变化
- **对话后心情会变**：开心、兴奋、平静、困惑等
- **状态会更新**：继续交流、思考、工作、休息等
- **真实反应**：基于对话内容和NPC性格

### 3. 个性化评估
- **考虑NPC性格**：外向/内向、友善/直率、开放/保守
- **考虑兴趣匹配**：提到NPC感兴趣的话题会加分
- **考虑当前关系**：陌生人和好友的评判标准不同

## AI评估流程

### 1. 构建评估提示

AI会收到以下信息：

```
你是[NPC名字]，一个真实的人。

【你的背景】
- 职业：创作者
- 兴趣：商业、创作、科技
- 性格：外向性 70/100 (外向)，宜人性 60/100 (友善)

【当前关系】
- 关系类型：熟人
- 好感度：25/100 (有好感)
- 熟悉度：35/100 (比较熟悉)
- 当前心情：😊 开心

【对话内容】
对方说："你最近在做什么创作？"
你回复："我最近在写一个关于AI的故事..."

【评估任务】
请评估这次对话，给出：
1. 好感度变化（-10到+10）
2. 心情变化
3. 状态变化
4. 评判理由
5. 情绪反应
```

### 2. AI分析对话

AI会考虑：
- **话题匹配**：对方问到了我感兴趣的"创作"话题 ✓
- **态度真诚**：对方表现出真实的兴趣
- **对话深度**：不是简单的寒暄，而是有深度的提问
- **性格契合**：我是外向友善的人，喜欢分享

### 3. 给出评估结果

```json
{
  "affinityChange": 3,
  "newMood": "excited",
  "newActivityStatus": "chatting",
  "reasoning": "对方问到了我感兴趣的创作话题，而且态度很真诚，让我感觉很舒服。",
  "emotionalResponse": "我感到很兴奋，想继续聊下去。"
}
```

## 与脚本评估的对比

### 脚本评估（旧方案）
```javascript
// 冷冰冰的规则
if (message.includes('创作')) {
  affinityChange += 2;
}
if (message.length > 20) {
  affinityChange += 1;
}
// 结果：+3，没有理由，没有感情
```

### AI评估（新方案）
```
AI理解：
- 对方提到了我的兴趣
- 态度真诚
- 想深入了解
- 让我感到被认可

结果：+5
理由："对方问到了我感兴趣的话题，而且态度很真诚"
感受："我感到很开心，想继续聊下去"
```

## 回退机制

如果AI评估失败（网络问题、token不足等），系统会使用简单的规则评估：

```typescript
// 回退评估
- 话题匹配：+2
- 积极词汇：+2
- 消极词汇：-3
- 消息长度：+1
```

但这只是保底方案，主要还是依赖AI评估。

- 考虑缓存策略

### 2. 响应时间
- AI评估需要1-2秒

- 可以异步更新

### 3. 错误处理

- 记录错误日志
