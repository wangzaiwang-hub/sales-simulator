## 关键检查点

### 检查点1：后端是否找到记录？
- 查看后端日志："✅ 过滤后有 X 条相关记录"
- 如果X = 0，说明过滤条件有问题
- 如果X > 0，说明后端找到了记录

### 检查点2：前端是否收到记录？
- 查看前端日志："🔵 前端：收到 X 条历史记录"
- 如果X = 0，说明后端返回了空数组
- 如果X > 0，说明前端收到了数据

### 检查点3：前端是否转换成功？
- 查看前端日志："🔵 前端：转换后有 Y 条消息"
- Y应该是X的2倍（每条记录包含用户消息和NPC回复）

### 检查点4：前端是否设置到state？
- 查看前端日志："✅ 前端：聊天历史已设置到state"
- 如果看到这个，说明已经设置成功

### 检查点5：聊天窗口是否显示？
- 查看聊天窗口
- 应该能看到历史消息

## 可能的问题

### 问题1：后端过滤后有0条记录

**原因**：ID不匹配

**检查**：
1. 查看后端日志中的userId和targetUserId
2. 在Supabase SQL Editor中运行：
```sql
SELECT "userId", "targetUserId", message, reply 
FROM "ChatMessage" 
LIMIT 10;
```
3. 对比ID是否匹配

### 问题2：前端收到0条记录

**原因**：后端返回空数组

**检查**：
1. 打开浏览器开发者工具
2. 切换到Network标签
3. 查找`chat-history`请求
4. 查看Response内容

### 问题3：前端收到记录但不显示

**原因**：
- state设置失败
- 或者渲染逻辑有问题

**检查**：
1. 查看前端日志，确认"✅ 前端：聊天历史已设置到state"
2. 在Console中输入：
```javascript
// 查看chatMessages
console.log(chatMessages);
```

### 问题4：显示空白

**原因**：
- 消息格式不对
- 或者CSS问题

**检查**：
1. 查看前端日志中的消息数组
2. 确认消息格式正确
