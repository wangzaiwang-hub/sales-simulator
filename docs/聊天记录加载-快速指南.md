# 聊天记录加载 - 快速指南

## 🎯 问题

你反馈说：刷新页面后聊天记录不显示，感觉没有记录到数据库里面。

## ✅ 解决方案

我已经完成了聊天历史加载功能的实现！现在系统会自动从数据库加载聊天记录。

## ⚠️ 重要：你需要做的事

### 第一步：运行数据库迁移脚本

聊天记录功能需要一个新的数据库表。你需要在 Supabase 中运行迁移脚本：

1. 打开 **Supabase Dashboard**
2. 进入 **SQL Editor**
3. 打开文件 `backend/migrations/add-chat-history.sql`
4. 复制全部内容
5. 粘贴到 SQL Editor 中
6. 点击 **Run** 运行

**如果不运行这个脚本，聊天记录功能将无法工作！**

### 第二步：测试功能

1. 与任意NPC聊天几句
2. 刷新页面（按F5）
3. 再次打开与该NPC的聊天窗口
4. 你应该能看到之前的聊天记录

## 📋 功能说明

### 现在的工作流程

```
1. 你与NPC聊天
   ↓
2. 聊天记录自动保存到数据库 ✅
   ↓
3. 聊天记录显示在前端 ✅
   ↓
4. 你刷新页面
   ↓
5. 打开聊天窗口时自动加载历史记录 ✅
   ↓
6. 聊天记录完整显示 ✅
```

### 保存的信息

每条聊天记录包含：
- ✅ 你的消息
- ✅ NPC的回复
- ✅ 好感度和熟悉度（当时的值）
- ✅ 好感度和熟悉度的变化
- ✅ AI的评判理由
- ✅ AI的情绪反应
- ✅ NPC的心情和状态变化

### 加载特点

- **自动加载**：打开聊天窗口时自动加载，无需手动操作
- **智能缓存**：已加载的记录不会重复加载
- **完整显示**：显示所有关系变化和AI评估信息
- **默认数量**：加载最近50条记录

## 🔧 如果遇到问题

### 问题1：聊天记录不显示

**可能原因**：数据库迁移脚本未运行

**解决方法**：
1. 打开 Supabase Dashboard
2. 进入 SQL Editor
3. 运行查询：`SELECT * FROM "ChatMessage" LIMIT 1;`
4. 如果报错"table does not exist"，说明需要运行迁移脚本
5. 运行 `backend/migrations/add-chat-history.sql`

### 问题2：控制台有错误

**解决方法**：
1. 按F12打开浏览器开发者工具
2. 查看Console标签中的错误信息
3. 截图发给我，我帮你分析

### 问题3：只显示新的聊天，旧的不显示

**这是正常的！**
- 旧的聊天记录是在功能实现前创建的，没有保存到数据库
- 从现在开始的所有聊天都会保存
- 刷新后都能看到
