## 📋 详细步骤

### 步骤1：运行数据库迁移脚本

1. 打开浏览器，访问 https://supabase.com/dashboard
2. 登录你的账号
3. 选择你的项目
4. 点击左侧菜单的 **SQL Editor**
5. 在你的代码编辑器中打开 `backend/migrations/add-chat-history.sql`
6. 复制全部内容（Ctrl+A, Ctrl+C）
7. 回到Supabase，粘贴到SQL Editor中（Ctrl+V）
8. 点击右下角的 **Run** 按钮
9. 等待执行完成
10. 看到 "✅ 聊天记录表创建完成！" 消息

### 步骤2：测试基本功能

1. 打开游戏页面（应该已经在运行）
2. 移动到任意NPC旁边
3. 点击NPC或按C键打开聊天窗口
4. 输入消息："你好"
5. 观察NPC的回复和关系变化
6. 再发送几条消息
7. 按F5刷新页面
8. 再次打开与该NPC的聊天窗口
9. **预期结果**：看到之前的所有对话

### 步骤3：测试多个NPC

1. 与第一个NPC（例如"李明"）聊天
2. 关闭聊天窗口
3. 与第二个NPC（例如"王芳"）聊天
4. 刷新页面
5. 分别打开两个NPC的聊天窗口
6. **预期结果**：各自的聊天记录都正确显示

### 步骤4：测试关系变化

1. 与NPC聊天
2. 观察每条回复下方的关系信息：
   - 关系类型（陌生人、熟人、朋友等）
   - 好感度（数值和文字描述）
   - 熟悉度（数值和文字描述）
   - 变化量（+3, +4等）
3. 刷新页面
4. 再次打开聊天
5. **预期结果**：历史记录中显示之前的关系变化

### 步骤5：测试AI评估

1. 与NPC聊天
2. 观察每条回复下方的AI评估：
   - 💭 情绪反应
   - 📝 评判理由
3. 刷新页面
4. 再次打开聊天
5. **预期结果**：历史记录中显示AI的评估信息

## 🐛 如果遇到问题

### 问题1：运行迁移脚本时报错

**错误信息**：`relation "User" does not exist`

**原因**：User表不存在

**解决**：先运行 `backend/init-database.sql` 创建基础表

---

**错误信息**：`permission denied`

**原因**：权限不足

**解决**：确保你是项目的所有者或管理员

---

**错误信息**：`syntax error`

**原因**：SQL语法错误

**解决**：确保复制了完整的SQL文件内容

### 问题2：聊天记录不显示

**检查步骤**：

1. 打开浏览器开发者工具（F12）
2. 切换到Console标签
3. 查看错误信息
4. 截图发给我

**常见原因**：

- 数据库迁移脚本未运行
- API调用失败
- 权限问题
- 网络问题

### 问题3：显示的记录不完整

**可能原因**：

- 旧的聊天记录（功能实现前）没有保存到数据库
- 这是正常的

**解决**：

- 从现在开始的所有聊天都会保存
- 旧记录无法恢复
