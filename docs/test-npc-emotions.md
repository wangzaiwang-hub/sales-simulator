# NPC情绪系统测试指南

## 测试步骤

### 1. 运行数据库迁移（如果还没运行）

在Supabase Dashboard → SQL Editor 中运行：
```sql
-- 复制 backend/migrations/add-npc-personality.sql 的内容并执行
```

### 2. 访问游戏

打开浏览器访问：https://192.168.110.169:3443/game

### 3. 观察NPC状态

你应该能看到：

#### 状态显示
- NPC头顶有两行文字
- 第一行：情绪表情 + 状态文本（如：😊 交流中）
- 第二行：NPC名字
- 状态标签有彩色背景

#### 不同状态示例
- 😊 闲逛中（灰色）
- 🤔 思考中（蓝色）
- 😴 忙碌中（琥珀色）
- 😊 交流中（绿色）
- 🤩 工作中（紫色）
- 😌 休息中（青色）
- 😠 争吵中（红色）
- 😊 开心聊天（琥珀色）
- 🤔 沉思中（靛蓝色）

### 4. 测试社交互动

#### 测试1：和不同NPC聊天
1. 靠近一个NPC
2. 按C键或点击NPC
3. 观察反应：
   - 有些NPC会接受聊天（打开对话框）
   - 有些NPC会拒绝，并给出理由

#### 测试2：观察拒绝理由
可能的拒绝理由：
- "我现在想一个人静静..."
- "抱歉，我现在有点忙..."
- "我现在心情不太好，改天再聊吧。"
- "我正在和别人说话呢。"
- "不好意思，我不太想聊天。"

#### 测试3：观察NPC之间的互动
- 等待一段时间
- 观察两个NPC靠近时的状态变化
- 可能看到：
  - 两个NPC都变成"交流中"
  - 两个NPC都变成"争吵中"（如果性格不合）
  - 两个NPC都变成"开心聊天"（如果都心情好）

### 5. 检查数据库中的性格数据

```bash
cd backend
node check-database.js
```

应该能看到每个用户的：
- personalityTraits（性格特征）
- currentMood（当前情绪）
- activityStatus（活动状态）

## 预期行为

### 外向的NPC
- 更容易接受聊天
- 可能主动靠近其他NPC
- 状态多为"交流中"或"开心聊天"
- 情绪多为😊或🤩

### 内向的NPC
- 经常拒绝聊天
- 独自在一个区域活动
- 状态多为"思考中"或"工作中"
- 情绪多为😌或🤔

### 宜人性低的NPC
- 可能随机拒绝聊天
- 和其他NPC相遇时可能"争吵中"
- 拒绝理由比较直接

### 神经质高的NPC
- 心情波动大
- 可能显示😰或😕
- 压力大时拒绝社交

## 故障排查

### 问题1：看不到状态标签

检查：
1. 前端是否正确重启
2. 浏览器控制台是否有错误
3. NPC数据是否包含情绪字段

解决：
```bash
# 重启前端
cd frontend
npm run dev
```

### 问题2：所有NPC都接受聊天

原因：数据库中没有性格数据

解决：
1. 运行数据库迁移脚本
2. 或者手动更新用户性格：
```sql
UPDATE "User" 
SET "personalityTraits" = jsonb_build_object(
    'openness', 50,
    'conscientiousness', 50,
    'extraversion', 30,  -- 设为30让NPC更内向
    'agreeableness', 50,
    'neuroticism', 50
)
WHERE id = '你的用户ID';
```

### 问题3：状态不变化

原因：前端还没实现动态更新

当前版本：
- 状态在NPC生成时确定
- 需要刷新页面才能看到新状态

未来改进：
- 实时更新NPC情绪
- 根据互动动态改变状态

### 问题4：后端报错

检查：
```bash
# 查看后端日志
cd backend
# 检查终端输出
```

常见错误：
- 缺少字段 → 运行数据库迁移
- 类型错误 → 检查TypeScript编译
