# 测试：聊天消息闪烁修复

## 问题

你反馈：发送消息后，AI回复了，但是界面上消息一闪而过。

## 我做的修复

修改了 `frontend/src/app/game/page.tsx`：

1. **移除了chatMessages依赖**
   - 之前useEffect依赖`[selectedNpcId, chatMessages]`
   - 现在只依赖`[selectedNpcId]`
   - 避免因消息更新触发重新加载

2. **添加了useRef跟踪**
   - 使用`loadedHistoryRef`记录已加载的NPC
   - 避免重复加载历史记录
   - 不会触发重新渲染

## 技术细节

### 修复前的问题

```typescript
// 问题代码
useEffect(() => {
  // ...
}, [selectedNpcId, chatMessages]); // ❌ chatMessages依赖导致问题
```

当发送新消息时：
1. `chatMessages`更新
2. 触发useEffect
3. useEffect可能重新加载历史
4. 导致消息闪烁或丢失

### 修复后的代码

```typescript
// 修复后
const loadedHistoryRef = useRef<Set<string>>(new Set());

useEffect(() => {
  if (!selectedNpcId) return;
  
  // 检查是否已加载
  if (loadedHistoryRef.current.has(selectedNpcId)) {
    return;
  }
  
  // 检查是否有消息
  if (chatMessages[selectedNpcId]?.length > 0) {
    loadedHistoryRef.current.add(selectedNpcId);
    return;
  }
  
  // 加载历史...
  
  // 标记为已加载
  loadedHistoryRef.current.add(selectedNpcId);
}, [selectedNpcId]); // ✅ 只依赖selectedNpcId
```

现在：
1. 只在切换NPC时触发
2. 使用useRef跟踪加载状态
3. 不会因消息更新而重新加载
