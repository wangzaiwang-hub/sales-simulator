## 🔧 故障排除

### 问题1：聊天记录不显示

**检查步骤**：

1. 打开浏览器开发者工具（按 F12）
2. 切换到 **Console** 标签
3. 查看是否有错误信息

**可能原因**：

- 数据库迁移脚本未运行
- API调用失败
- 权限问题

**解决方法**：

1. 确认已运行 `backend/migrations/add-chat-history.sql`
2. 在Supabase中运行查询：`SELECT * FROM "ChatMessage" LIMIT 1;`
3. 如果报错"table does not exist"，说明需要运行迁移脚本

### 问题2：只显示新的聊天

**这是正常的！**

- 旧的聊天记录是在功能实现前创建的
- 没有保存到数据库
- 从现在开始的所有聊天都会保存

### 问题3：加载很慢

**原因**：

- 默认加载最近50条记录
- 如果记录很多，可能需要一些时间

**优化**：

- 系统会缓存已加载的记录
- 不会重复加载

## 附录：数据库表结构

```sql
CREATE TABLE "ChatMessage" (
    id TEXT PRIMARY KEY,
    userId TEXT NOT NULL,              -- 发送者ID
    targetUserId TEXT NOT NULL,        -- 接收者ID（NPC）
    message TEXT NOT NULL,             -- 用户消息
    reply TEXT,                        -- NPC回复
    source TEXT,                       -- 回复来源
    affinity INTEGER,                  -- 好感度
    familiarity INTEGER,               -- 熟悉度
    relationshipType TEXT,             -- 关系类型
    affinityChange INTEGER,            -- 好感度变化
    familiarityChange INTEGER,         -- 熟悉度变化
    aiReasoning TEXT,                  -- AI评判理由
    aiEmotionalResponse TEXT,          -- AI情绪反应
    npcMoodBefore TEXT,                -- NPC对话前心情
    npcMoodAfter TEXT,                 -- NPC对话后心情
    npcStatusBefore TEXT,              -- NPC对话前状态
    npcStatusAfter TEXT,               -- NPC对话后状态
    createdAt TIMESTAMP DEFAULT NOW()  -- 创建时间
);
```
