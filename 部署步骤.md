## 第二步：部署前端到 Vercel

### 1. 访问 Vercel
打开浏览器，访问：https://vercel.com

### 2. 登录/注册
- 点击右上角 "Sign Up"
- 使用 GitHub 账号登录（推荐）

### 3. 创建新项目
- 点击 "Add New..."
- 选择 "Project"
- 找到你的仓库：`wangzaiwang-hub/sales-simulator`
- 点击 "Import"

### 4. 配置项目

#### 4.1 设置 Root Directory
- 在 "Configure Project" 页面
- 找到 "Root Directory"
- 点击 "Edit"
- 输入：`frontend`
- 点击 "Continue"

#### 4.2 添加环境变量
- 在 "Environment Variables" 部分
- 添加以下变量（使用第一步获取的后端域名）：

```bash
NEXT_PUBLIC_SECONDME_CLIENT_ID=3fd385c8-95d8-4ec9-a547-16e104da067f
NEXT_PUBLIC_API_URL=https://你的后端域名.railway.app
BACKEND_URL=https://你的后端域名.railway.app
```

**注意：** 先不要填写 `NEXT_PUBLIC_SECONDME_REDIRECT_URI`，等部署完成后再填。

### 5. 开始部署
- 点击 "Deploy"
- 等待部署完成（约 2-3 分钟）

### 6. 获取前端域名
- 部署成功后，Vercel 会显示你的域名
- 例如：`https://sales-simulator-xxx.vercel.app`

**📝 记下这个域名，后面需要用到！**

## 第三步：更新环境变量

现在你有了前后端的域名，需要更新环境变量。

### 3.1 更新 Railway 环境变量

回到 Railway 控制台：

1. 进入你的项目
2. 点击 "Variables" 标签
3. 添加/更新以下变量（使用第二步获取的前端域名）：

```bash
BASE_URL=https://你的后端域名.railway.app
FRONTEND_URL=https://你的前端域名.vercel.app
CORS_ORIGIN=https://你的前端域名.vercel.app
SECONDME_REDIRECT_URI=https://你的前端域名.vercel.app/auth/callback
```

4. 点击 "Deployments" 标签
5. 点击最新的部署
6. 点击 "Redeploy"

### 3.2 更新 Vercel 环境变量

回到 Vercel 控制台：

1. 进入你的项目
2. 点击 "Settings" 标签
3. 点击左侧 "Environment Variables"
4. 添加新变量：

```bash
NEXT_PUBLIC_SECONDME_REDIRECT_URI=https://你的前端域名.vercel.app/auth/callback
```

5. 点击 "Deployments" 标签
6. 点击最新的部署右侧的 "..."
7. 选择 "Redeploy"

## ⚠️ 常见问题

### 1. 部署失败
- 检查 Railway/Vercel 的日志
- 确认环境变量是否正确
- 确认 Root Directory 设置正确

### 2. CORS 错误
- 检查后端的 `CORS_ORIGIN` 是否设置为前端域名
- 确认前后端域名都是 HTTPS

### 3. OAuth 回调失败
- 检查 SecondMe 控制台的回调地址
- 确认 `SECONDME_REDIRECT_URI` 正确

### 4. 地图不显示
- 检查后端的静态资源是否正确部署
- 确认 Railway 部署了 `backend/resource` 目录

### 5. 数据库连接失败
- 检查 Supabase 连接字符串
- 确认服务密钥正确
